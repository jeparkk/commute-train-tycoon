# 칙칙폭폭 출퇴근 익스프레스 게임감 디벨롭 계획

## 현재 상태 진단

8단계까지 구현한 결과물은 기능 프로토타입으로는 의미가 있지만, 아직 플레이하고 싶은 게임처럼 보이지 않는다. 이유는 명확하다.

- 화면의 중심이 열차 객실이 아니라 카드형 UI와 버튼이다.
- 좌석, 매점, 장식이 실제 객실 안 물건처럼 보이지 않고 업그레이드 카드처럼 보인다.
- 레벨업을 해도 객실 풍경이 크게 변하지 않아 성장 체감이 약하다.
- 승객, 분실물, 역, 이동감 같은 출퇴근 열차 판타지가 거의 없다.
- 도트/일러스트 에셋이 실제로 적용되지 않아 앱 프로토타입 느낌이 강하다.
- 상점, 이동, 정산은 기능적으로는 있지만 게임 세계 안 이벤트처럼 연출되지 않는다.

따라서 다음 개발 방향은 기능 추가보다 "게임 화면의 재구성"이 우선이다. 목표는 사용자가 첫 화면을 봤을 때 바로 "귀여운 2D 열차 방치형 게임"이라고 느끼게 만드는 것이다.

## 최종 목표 화면

첫 화면은 하나의 2D 열차 객실이어야 한다.

상단에는 최소한의 재화 정보만 둔다. 중앙 대부분은 객실 내부가 차지한다. 객실에는 창문, 좌석, 손잡이, 바닥, 매점, 장식 슬롯, 승객, 마스코트가 한 장면 안에 배치된다. 사용자는 화면 속 물건을 직접 누르는 느낌으로 업그레이드한다.

UI는 다음 원칙을 따른다.

- 설명보다 시각적 상태가 먼저 보여야 한다.
- 버튼은 하단 네비게이션과 중요한 액션에만 쓴다.
- 객실 안 오브젝트가 곧 버튼이다.
- 자유 배치는 하지 않는다. 정해진 위치가 레벨에 따라 바뀐다.
- 레벨업은 수치 증가가 아니라 객실이 예뻐지는 변화로 보여준다.
- 모든 팝업은 게임 세계의 정산표, 티켓, 역 안내판 같은 형태로 보인다.

## 핵심 방향

### 1. 객실 장면을 게임의 메인 무대로 재설계

현재 `TrainCabin`은 카드형 오브젝트 3개가 들어간 패널에 가깝다. 다음 단계에서는 객실을 하나의 고정 비율 2D 씬으로 바꾼다.

구성:

- 배경: 열차 벽, 창문 3~4개, 바깥 풍경
- 전경: 바닥, 좌석, 매점, 장식 슬롯
- 캐릭터: 승객 2~4명, 마스코트
- 인터랙션 포인트: 좌석, 매점, 창가 장식, 벽 장식, 바닥 장식
- 상태 연출: 수익 발생, 승객 탑승, 분실물 반짝임, 이동 중 흔들림

기술 방향:

- `TrainCabin`을 카드 레이아웃에서 `Stack` 기반 씬으로 변경한다.
- 객실 전체는 `AspectRatio`로 고정해 모바일에서도 장면이 깨지지 않게 한다.
- 각 오브젝트는 고정 좌표 슬롯으로 배치한다.
- 레벨별 이미지 키를 `GameAssets`에서 관리한다.

### 2. 아트 스타일 확정

추천 스타일은 "부드러운 픽셀풍 2D 일러스트"다. 완전한 16비트 도트보다 모바일 방치형에 더 잘 맞는다.

완전 도트의 장점:

- 귀엽고 인디 게임 느낌이 강하다.
- 작은 리소스로도 통일감을 만들 수 있다.

완전 도트의 단점:

- 에셋 제작 규칙이 엄격하다.
- 해상도 대응이 까다롭다.
- 초보 AI 에셋 생성 결과가 들쭉날쭉할 수 있다.

추천하는 절충안:

- 외곽선은 또렷하게 둔다.
- 색면은 단순하게 둔다.
- 질감은 약하게만 넣는다.
- 픽셀 게임처럼 보이지만 실제 파일은 고해상도 PNG로 만든다.
- UI는 도트 폰트보다 둥글고 읽기 쉬운 모바일 폰트를 쓴다.

아트 키워드:

- cozy 2D train cabin
- cute commuter tycoon
- soft pixel art inspired
- clean mobile game asset
- fixed isometric side-view room
- warm pastel with teal, yellow, coral accents

피해야 할 것:

- 너무 어두운 지하철
- 실제 사진 느낌
- 과한 사이버펑크
- UI 카드가 화면을 덮는 구성
- 베이지/갈색만 많은 단조로운 팔레트

### 3. 에셋 우선순위

처음부터 모든 이미지를 만들 필요는 없다. 게임감에 가장 큰 영향을 주는 순서대로 만든다.

1. 객실 배경 1장
2. 좌석 레벨 1~3
3. 매점 레벨 1~3
4. 창문 바깥 풍경 2~3종
5. 장식 슬롯 3개, 각 2~3종
6. 승객 캐릭터 3종
7. 마스코트 1종
8. 분실물 반짝임 아이콘
9. 티켓/정산 팝업 배경
10. 상점/VIP 패스 배너

MVP 다음 단계에서는 레벨 20 전체 에셋이 필요하지 않다. 레벨 1, 2, 3만 시각적으로 확실히 다르게 만들고, 이후 레벨은 색상/소품 변화로 확장한다.

## 다음 개발 단계

### Step 9. 메인 객실 씬 리빌드

목표:

현재 카드형 객실을 실제 2D 열차 객실처럼 바꾼다.

작업:

- `TrainCabin`을 `Stack` 기반 씬으로 재작성
- 좌석, 매점, 장식 슬롯을 실제 객실 위치에 고정 배치
- 창문, 바닥, 손잡이, 안내판 등 열차 요소 추가
- 하단 버튼보다 객실 오브젝트 터치가 중심이 되게 수정
- 레벨업 가능 오브젝트에는 은은한 반짝임 표시

완료 기준:

- 스크린샷만 봐도 열차 객실로 보인다.
- 좌석과 매점이 카드가 아니라 물건처럼 보인다.
- 사용자가 어디를 눌러야 할지 말 없이 알 수 있다.

### Step 10. 임시 벡터/커스텀 페인터 에셋 적용

목표:

최종 PNG가 없어도 게임처럼 보이도록 Flutter 코드 기반 임시 2D 에셋을 만든다.

작업:

- 좌석, 매점, 창문, 손잡이, 마스코트를 간단한 코드 드로잉으로 구현
- 기존 아이콘 fallback 비중 축소
- 레벨 1~3에 따라 외형이 바뀌게 구성
- 나중에 PNG로 교체할 수 있도록 asset key 구조 유지

완료 기준:

- 아이콘 앱 느낌이 줄어든다.
- 에셋이 없어도 귀여운 게임 화면으로 보인다.
- PNG 교체가 쉬운 구조를 유지한다.

### Step 11. 승객과 수익 발생 연출 추가

목표:

방치형 게임의 생동감을 만든다.

작업:

- 승객 2~4명이 객실에 앉거나 서 있게 배치
- 일정 주기로 말풍선 또는 코인 팝업 표시
- 수익 발생 시 `+G` 텍스트가 오브젝트 위로 떠오르게 구현
- VIP 승객은 열차 매력도 조건으로 등장하도록 구조 설계

완료 기준:

- 가만히 둬도 화면이 살아있어 보인다.
- 골드가 늘어나는 이유가 시각적으로 보인다.

### Step 12. 꾸미기 UX를 객실 중심으로 변경

목표:

장식 관리가 상점 리스트가 아니라 객실 꾸미기처럼 느껴지게 만든다.

작업:

- 장식 슬롯을 객실 안에 직접 노출
- 빈 슬롯은 점선/반짝임으로 표시
- 슬롯 터치 시 해당 위치에 맞는 아이템만 보여주는 작은 선택 팝업 표시
- 구매 후 즉시 객실에 배치되는 연출 추가
- 업그레이드 시 외형이 더 풍성해지게 수정

완료 기준:

- 사용자가 "내 객실을 꾸민다"고 느낀다.
- 드래그 없이도 배치 감각이 생긴다.

### Step 13. 이동/오프라인 정산을 게임 세계 연출로 변경

목표:

정산 팝업을 일반 앱 바텀시트가 아니라 열차 게임 이벤트처럼 만든다.

작업:

- 오프라인 정산 팝업을 역무원 정산표 디자인으로 변경
- 이동 보너스는 "도착역 리포트" 형태로 변경
- GPS 기준 위치 저장은 "출발역 등록"으로 표현
- 이동 보상은 티켓 스탬프/노선도 연출로 보여주기

완료 기준:

- 정산이 기능 팝업이 아니라 게임 이벤트처럼 느껴진다.

### Step 14. 첫 세션 온보딩 추가

목표:

처음 실행한 유저가 30초 안에 재미를 이해하게 만든다.

작업:

- 첫 접속 시 좌석 업그레이드 유도
- 매점 해금 또는 장식 구매까지 짧은 목표 제공
- 첫 이동 보너스 기준 위치 저장 안내
- 튜토리얼은 긴 설명 대신 손가락 표시/하이라이트로 처리

완료 기준:

- 신규 유저가 헤매지 않는다.
- 첫 1분 안에 성장, 꾸미기, 이동 보상의 맛을 본다.

### Step 15. 아트 에셋 제작 파이프라인 확정

목표:

AI 이미지 생성과 앱 적용을 반복 가능한 방식으로 만든다.

작업:

- `assets/images/slots`, `decorations`, `characters`, `backgrounds` 구조 정리
- 에셋 크기 규격 정하기
- 배경 투명 PNG 기준 확정
- 레벨별 naming rule 확정
- AI 프롬프트 템플릿 작성

권장 크기:

- 객실 배경: 1600x900
- 슬롯 오브젝트: 512x512
- 승객/마스코트: 512x512
- 장식: 384x384
- UI 배너: 1200x400

완료 기준:

- 새 이미지를 넣으면 코드 수정 없이 또는 최소 수정으로 교체 가능하다.

## 화면 구조 재설계안

메인 화면:

```text
┌────────────────────────────┐
│ 골드 / WP / VIP 작은 표시   │
├────────────────────────────┤
│                            │
│      2D 열차 객실 씬        │
│                            │
│  창문  창문  창문            │
│  좌석   승객   매점          │
│  장식   마스코트  분실물     │
│                            │
├────────────────────────────┤
│ 객실 | 이동 | 상점           │
└────────────────────────────┘
```

객실 안 터치 대상:

- 좌석: 골드로 업그레이드
- 매점: 골드로 업그레이드
- 창가 장식: 구매/교체/업그레이드
- 벽 장식: 구매/교체/업그레이드
- 바닥 장식: 구매/교체/업그레이드
- 분실물: 즉시 보상
- 마스코트: 버프 충전

## 우선순위 결정

가장 먼저 해야 할 일은 GPS, 광고, BM을 더 붙이는 것이 아니다. 이미 기능 구조는 충분히 있다. 이제 게임으로 보이게 만들어야 한다.

추천 우선순위:

1. Step 9 메인 객실 씬 리빌드
2. Step 10 임시 2D 에셋 적용
3. Step 11 승객/수익 연출
4. Step 12 꾸미기 UX 재설계
5. Step 13 정산 팝업 게임화
6. Step 14 온보딩
7. Step 15 에셋 파이프라인

이 순서가 좋은 이유:

- 첫 화면의 인상이 바로 좋아진다.
- 게임답지 않다는 가장 큰 문제를 먼저 해결한다.
- 이후 GPS, 광고, VIP도 게임 세계 안에 자연스럽게 녹일 수 있다.
- 실제 PNG 에셋이 늦어져도 코드 기반 임시 아트로 게임감을 검증할 수 있다.
